TaskSpawner

TaskSpawners connect cluster workflows to external engineering lifecycles. They act as automated triggers that poll systems for changes and instantiate individual Tasks when matching filters occur.

  • Trigger Sources: Event channels like GitHub Webhooks, Jira issue states, and Cron schedules.
  • Rate Limiting: Limits spending and resource usage via strict maxConcurrency definitions.
  • Lifecycle Binding: Map standard issue triggers to structured Task template pipelines.
Task

A Task specifies a single autonomous agent execution pod. It outlines the specific LLM model to request, container configurations, instruction prompt statements, and references.

  • Host Isolation: Spawns in an ephemeral, completely secure Kubernetes Pod environment.
  • Deterministic Outputs: Captures logs, commit SHAs, git branch outputs, and PR listings in status.
  • Action Chains: Sequence agent logic in pipelines using the dependsOn keyword.
Workspace

Workspaces outline the target code repository where actions occur. They securely handle private credentials, target branches, custom setup actions, and repository caching configs.

  • Secure Credentials: Safely connects via fine-grained GitHub PATs or Enterprise App certs.
  • Pre-flight Hooks: Run custom bootstrap operations (e.g. go mod download) prior to startup.
  • Branch Controls: Automated local git status checkouts and remote tracking bindings.
AgentConfig

AgentConfigs package specialized system instructions, static rules, model limitations, and Model Context Protocol (MCP) server endpoints to extend agent powers.

  • Standardized Rules: Injects static guidelines (like CLAUDE.md) to enforce team code styling.
  • Custom Skills: Bundles custom scripts and capabilities to perform advanced unit runs.
  • MCP Toolkits: Connects agents directly to cluster resources, databases, or REST APIs.

CLI Reference Console

kelos install

Deploy the Kelos controller deployment, CRDs, and necessary RBAC policies into the active Kubernetes cluster context.

$ kelos install ℹ️ Active cluster: "gke-dev-cluster" ℹ️ Deploying CRDs... [Done] ℹ️ Creating namespace "kelos-system"... [Done] ℹ️ Installing RBAC permissions... [Done] ℹ️ Deploying controller manager... [Done] ✅ Kelos installed successfully!

kelos init

Generate a default settings file locally under ~/.kelos/config.yaml to store developer access keys and repository authentication.

$ kelos init ✅ Configuration folder created: ~/.kelos/ ✅ Generated default configuration file: ~/.kelos/config.yaml ℹ️ Next: edit config.yaml with API tokens & repo info.

kelos run

Instantiate a temporary task pod on the cluster to execute an on-demand instruction directly from your current workspace.

$ kelos run -p "Fix the flaky unit test" -w 🚀 Creating Task resource "task-r8x2q"... [15:20:00] Pod Status: Pending ➔ Running [15:20:02] Workspace: Cloning repo... [Done] [15:20:06] Agent: Running Claude Code agent... [15:20:14] Agent: Editing file utils_test.go [Done] ✅ Task task-r8x2q Succeeded! PR created: #45

kelos logs

Stream the standard stdout and stderr logs of the active agent running inside the cluster pod for real-time auditing.

$ kelos logs task-r8x2q -f [15:20:06] [Kelos] Starting agent Claude Code in container... [15:20:08] [Claude Code] System rules loaded. Scan matches found. [15:20:11] [Claude Code] Modifying internal/utils_test.go: - time.Sleep(5 * time.Second) + time.Sleep(10 * time.Millisecond) ✅ Stream finished. Pod task-r8x2q-pod terminated (Completed).

kelos suspend taskspawner issue-autofixer

Instruct the active controller to temporarily pause a TaskSpawner. Any matching events (e.g. issues created) will be bypassed until it is resumed.

$ kelos suspend taskspawner issue-autofixer ✅ TaskSpawner "issue-autofixer" suspended. ℹ️ Events will be ignored. To resume, run: "kelos resume taskspawner issue-autofixer".
